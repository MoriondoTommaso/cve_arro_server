# Repository Cleanup — Manual Steps Required

The `.gitignore` prevents new binary artefacts from being committed, but **already-tracked files must be explicitly removed** with `git rm --cached`. Run the steps below once on a clean checkout of `main`.

## Files to remove from git tracking

| Path | Reason |
|------|--------|
| `arrowspace_index/main--matrix/` | Runtime-generated Zarr index (regenerated by `POST /datasets/{id}/index`) |
| `_arrowspace_index/example_data--matrix/` | Runtime-generated Zarr index |
| `_arrowspace_index/main/` | Runtime-generated Zarr index |
| `embeddings_nomic_structured_768d_raw.zarr/` | Binary 768-d embedding corpus (regenerate via `scripts/nomic_corpus_embs.py`) |
| `output_prompt_nomic768/` | Ablation output artefacts (regenerate via scripts) |

## Commands

```bash
# 1. Untrack all binary/generated artefact directories
git rm -r --cached arrowspace_index/main--matrix/
git rm -r --cached _arrowspace_index/example_data--matrix/
git rm -r --cached _arrowspace_index/main/
git rm -r --cached embeddings_nomic_structured_768d_raw.zarr/
git rm -r --cached output_prompt_nomic768/

# 2. Commit the removal (files stay on disk, just leave git history)
git commit -m "chore: untrack committed binary artefacts (zarr index, embeddings, outputs)"

# 3. Push
git push origin main
```

> ⚠️ **Recommended follow-up**: run `git filter-repo` or BFG Repo-Cleaner to also strip these blobs from full git history and reduce clone size.
> ```bash
> # Example with BFG (removes blobs > 1MB from history)
> bfg --strip-blobs-bigger-than 1M
> git reflog expire --expire=now --all
> git gc --prune=now --aggressive
> git push --force
> ```

## Reproducing artefacts after removal

```bash
# Zarr embeddings corpus
uv pip install -e '.[nlp]'
uv run scripts/nomic_corpus_embs.py

# ArrowSpace graph-Laplacian index (server must be running)
curl -X POST http://localhost:8000/api/datasets/main/index

# Example data
uv run scripts/make_example_data.py
```
